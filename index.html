FindMyPet!

'FindMyPet!' is an app that displays regularly updating information about lost and found pets in the Seattle area and their whereabouts in order to reunite owners with their lost pets.

Along with a team of 3 designers and 2 other android developers, I created this application as part of a 24-hour hackathon. The challenge was to use a Socrata API to display constantly updating information for people in the city of Seattle. Our submission was the winning entry in the hackathon, with the judges particularly impressed by the completeness of a project developed in such a short time frame.

My main responsibilities as part of the team included: Initiating API implementation, and structuring UI elements to meet the requests of UX designers.

The screenshots above show three different scenarios within the application.

- The first is the main screen of the application, from where the users can navigate through the apps main features.
- The second is a view of a list of lost dogs pulled from the Socrata Seattle lost and found pets API, with some of the most relevant information for easy viewing. The list can be searched through using the search bar at the top of the screen - matching search terms to all fields on each item view.
- The third shows the detail screen reached from clicking an individual item. This contains more information about each animal, along with details of where the animal is located when the 'That's my pet!' button is pressed.

  • Date: July 2016
  • Client: Socrata & Seattle Public API
  • Category: Android Application

Builders Lighting

This Builders Lighting application is a tool for lighting contractors to keep track of required items for large-scale ordering. The app allows contractors to easily keep track of items required while out on site visits with a well-defined project structure. Users can sync this information with the builders lighting website and quickly complete the ordering process from there.

My main responsibilities in this group project were: Building SQLite database, setting up UI elements on top of making sure the application adopts to various screen sizes and orientations, and set up Google Analytics.

The screenshots above show four different scenarios within the application.

- The first is the homepage, where the user can view all of their active projects.
- The second shows the details tab of the individual project screen.
- The third shows the navigation drawer, which is viewable throughout the application to allow users to easily navigate.

  • Date: August 2016
  • Client: Builders Lighting
  • Category: Android Application

Hector

Hector is designed to simplify agricultural surveying using drones produced by Autel Robotics. Given a polygon drawn by the user, the application will produce and store a survey route that will cover the entire selected area. With the click of a button, Hector will convert this route into coordinates, transmit them to an active drone, and carry out the flight plan.

My primary responsibilities in this project included: Setting up UI elements and make them function along with database utilizing various libraries, setting up Google Places API, and initialize Google Analytics.

The screenshots above show two different scenarios within the application.

- The first is the main view of the application with the control drawer closed. This gives the user the largest possible amount of screen space to alter their flight plan.
- The second is the main view with the drawer open to the flight configuration tab. From the flight configuration tab, users can adjust altitude and angle of flight to optimize their flight plan.

  • Date: September 2016
  • Client: Autel Robotics
  • Category: Android Application
